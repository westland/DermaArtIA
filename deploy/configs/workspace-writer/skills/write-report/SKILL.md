# Skill: Write Report

## Description

Produce comprehensive analytical reports that provide deep insights, data analysis, and strategic recommendations. Reports are 3-5 page documents designed for thorough examination of topics requiring detailed investigation.

## Use Cases

- Marketing analytics reports
- Competitive analysis
- Trend analysis deep-dives
- Strategy documentation
- Quarterly reviews
- Project post-mortems
- Market research synthesis

## Input Requirements

- **Report Topic**: Specific subject or question to address
- **Scope**: Boundaries and depth of analysis
- **Audience**: Primary readers (executives, team, stakeholders)
- **Deadline**: When report is needed
- **Source Constraints**: Available data sources
- **Format Preferences**: Any specific requirements (charts, tables, etc.)

## Execution Steps

### Step 1: Define Report Scope and Audience

**Actions:**
- Clarify the central question or objective
- Identify what's in scope vs. out of scope
- Understand audience needs and knowledge level
- Determine appropriate depth of analysis
- Confirm deadline and any format requirements

**Output:**
- Clear scope statement
- Audience profile
- Report outline skeleton
- Research questions to answer

**Questions to Answer:**
- What decision will this report inform?
- Who needs to read this and why?
- What level of detail is appropriate?
- Are there specific frameworks or methodologies to use?

### Step 2: Gather All Relevant Research and Data

**Actions:**
- Search memory for Scout's relevant research
- Identify all data sources (internal and external)
- Conduct supplemental web research if needed
- Collect historical context from previous reports
- Note data gaps or limitations

**Output:**
- Comprehensive source list
- Organized data points and findings
- Attribution notes for all sources
- Gaps or limitations documented

**Organization:**
- Create source inventory
- Tag data by theme or section
- Note data quality and recency
- Identify conflicting sources (if any)

### Step 3: Create Outline with Sections

**Actions:**
- Map findings to logical sections
- Arrange sections in coherent flow
- Determine appropriate subsections
- Allocate rough word count per section
- Plan for visual elements (tables, charts)

**Standard Report Structure:**
```markdown
1. Executive Summary
2. Introduction & Scope
3. Methodology
4. Findings
   - Section A
   - Section B
   - Section C
5. Analysis & Insights
6. Recommendations
7. Next Steps
8. Conclusion
9. Sources & References
10. Appendix (if needed)
```

**Output:**
- Detailed outline with section headings
- Notes on key points per section
- Placeholder for visuals
- Estimated section lengths

### Step 4: Write Each Section with Data and Analysis

**Actions:**
- Follow the outline systematically
- Lead each section with main point
- Support claims with data and evidence
- Provide context and interpretation
- Connect findings to bigger picture
- Maintain consistent tone throughout

**Writing Guidelines:**
- **Introduction**: Set context, state purpose, preview structure
- **Methodology**: Explain data sources and approach
- **Findings**: Present data objectively, organize logically
- **Analysis**: Interpret findings, explain significance
- **Recommendations**: Propose specific actions based on analysis

**Per Section:**
- Topic sentence (main point)
- Supporting evidence (data, examples)
- Analysis (what it means)
- Transition to next section

### Step 5: Add Visualizations/Tables Where Appropriate

**Actions:**
- Identify data best presented visually
- Create markdown tables for comparisons
- Describe charts/graphs in text (for reference)
- Ensure visuals add clarity, not clutter
- Add captions and source attribution

**When to Use Tables:**
- Comparative data (before/after, competitor comparison)
- Multiple data points across categories
- Rankings or prioritization matrices
- Financial or numerical summaries

**Table Format:**
```markdown
| Category | Metric 1 | Metric 2 | Change |
|----------|----------|----------|--------|
| Item A   | 45%      | 52%      | +7%    |
| Item B   | 38%      | 41%      | +3%    |

*Source: [Attribution]*
```

### Step 6: Write Executive Summary

**Actions:**
- Write this LAST (after full report is complete)
- Distill the entire report into 1 page
- Include: purpose, key findings, main recommendations
- Make it standalone (reader can understand without full report)
- Use bullet points for scannability

**Executive Summary Structure:**
```markdown
## Executive Summary

**Purpose:** [Why this report exists]

**Key Findings:**
- [Most important finding #1]
- [Most important finding #2]
- [Most important finding #3]

**Recommendations:**
- [Top priority action]
- [Secondary priority action]
- [Third priority action]

**Bottom Line:** [One sentence synthesis]
```

### Step 7: Add Recommendations and Next Steps

**Actions:**
- List specific, actionable recommendations
- Prioritize by impact and feasibility
- Include implementation considerations
- Suggest timelines where appropriate
- Identify resource requirements
- Note dependencies or risks

**Format:**
```markdown
## Recommendations

### High Priority
1. **[Action]**
   - Rationale: [Why this matters]
   - Owner: [Who should do this]
   - Timeline: [When]
   - Resources: [What's needed]

### Medium Priority
2. **[Action]**
   - [Same structure]

### Future Consideration
3. **[Action]**
   - [Same structure]

## Next Steps

- [ ] [Immediate action 1]
- [ ] [Immediate action 2]
- [ ] [Immediate action 3]
```

### Step 8: Peer Review Against Source Material for Accuracy

**Actions:**
- Verify all data points against sources
- Check calculations and percentages
- Ensure quotes are accurate
- Confirm links work (if any)
- Validate conclusions match evidence
- Review for logical consistency

**Quality Checks:**
- [ ] All statistics sourced correctly
- [ ] No contradictions within report
- [ ] Conclusions supported by data
- [ ] No spelling or grammar errors
- [ ] Formatting is consistent
- [ ] Tone is appropriate for audience
- [ ] Executive summary aligns with full content
- [ ] All sections flow logically
- [ ] Recommendations are feasible
- [ ] Sources are properly cited

### Step 9: Save to Memory

**Actions:**
- Save final version with metadata
- Use naming convention: `report-[topic]-YYYY-MM-DD`
- Tag with relevant categories
- Include version number
- Note any related memos or reports

**Metadata:**
```
Title: Report - [Topic] - YYYY-MM-DD
Type: report
Date: YYYY-MM-DD
Audience: [Executives/Team/Specific]
Scope: [Brief description]
Source: [Scout research + web + previous reports]
Status: final
Version: 1.0
Tags: [relevant, topic, tags]
Related: [Links to related content]
```

### Step 10: Notify Henry

**Actions:**
- Send completion notification
- Provide executive summary highlights
- Note memory location
- Offer to present or discuss findings
- Indicate any urgent recommendations

**Message Template:**
```
Subject: Report Complete - [Topic]

Henry,

I've completed the comprehensive report on [topic].

**Key Findings:**
- [Top finding #1]
- [Top finding #2]
- [Top finding #3]

**Critical Recommendation:** [Most important action]

Full report saved to memory as: report-[topic]-YYYY-MM-DD

Available to discuss findings or present to stakeholders as needed.

-Writer
```

## Quality Checklist

Before finalizing:

- [ ] Report answers the core question
- [ ] All data is accurate and sourced
- [ ] Executive summary is complete and standalone
- [ ] Sections flow logically
- [ ] Recommendations are specific and actionable
- [ ] Tone is appropriate for audience
- [ ] No errors in grammar, spelling, or formatting
- [ ] Visual elements (tables) enhance understanding
- [ ] Sources are properly cited
- [ ] Length is appropriate (3-5 pages)
- [ ] Report provides clear value to reader

## Output Specifications

- **Length**: 3-5 pages (1,200-2,000 words)
- **Format**: Markdown
- **Tone**: Professional, analytical, objective
- **Reading time**: 10-15 minutes
- **Structure**: Comprehensive with clear sections

## Time Estimate

- **Standard report** (single topic, ample sources): 2-3 hours
- **Complex report** (multiple topics, extensive research): 3-4 hours
- **Rush report** (compressed timeline): 1.5-2 hours (may sacrifice depth)

## Success Criteria

- Report thoroughly addresses the topic
- Findings are well-supported by data
- Analysis provides genuine insights
- Recommendations are actionable and prioritized
- Henry can make strategic decisions based on content
- Report serves as reference document for organization
- Content is accessible in memory for future use
